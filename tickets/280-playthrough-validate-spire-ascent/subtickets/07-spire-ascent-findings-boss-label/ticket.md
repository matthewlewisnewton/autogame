# Spire Ascent findings: preset-aware boss label and artifact regen

`game/validation/spire-ascent/findings.md` incorrectly reports `bossSpawned (annex_overseer): PASS` even though `run-summary.json` records `bossType: "spire_warden"` and the ticket requires confirming the Summit Warden stage boss. The shared `renderFindings` helper hard-codes the Training Caverns boss id and document title. Make findings preset-aware, regenerate the Spire Ascent validation tree, and tighten the artifact verifier so this mismatch cannot pass again.

## Acceptance Criteria

- `harness/validate/lib/findings.mjs` `renderFindings` accepts `bossType` (and uses it in the `bossSpawned` assertion line, e.g. `bossSpawned (spire_warden)`), and the document title reflects the preset (not the hard-coded `"Rooms validation findings"` string when `preset` is `spire-ascent`).
- `harness/validate/playthrough.mjs` passes `bossType` from the loaded preset into `renderFindings` inside `writeFullArtifacts` (and any other call sites).
- Implementer's **first action** after code changes: `cd game && pnpm validate:spire-ascent`. The npm script regenerates `game/validation/spire-ascent/findings.md` (do not hand-edit findings).
- `game/validation/spire-ascent/findings.md` contains `bossSpawned (spire_warden)` or `bossSpawned (Summit Warden)` with PASS when assertions pass — **not** `annex_overseer`.
- `cd game && pnpm validate:spire-ascent:check` exits `0`.
- `harness/validate/verify-spire-ascent-artifacts.mjs` rejects `findings.md` that still mentions `annex_overseer` or omits `spire_warden` / `Summit Warden` in the boss-spawned assertion line.
- Rooms preset (`pnpm validate:rooms` if present, or a focused unit/assertion on `renderFindings`) still renders `bossSpawned (annex_overseer)` — no regression for ticket 277 artifacts.
- `cd game && pnpm test:quick` passes.

## Technical Specs

- **`harness/validate/lib/findings.mjs`**
  - Add `bossType` (required when rendering boss assertion) and derive title from `preset` (e.g. `spire-ascent` → `"Spire Ascent validation findings"`, `rooms` → `"Rooms validation findings"`).
  - Replace hard-coded `formatAssertion('bossSpawned (annex_overseer)', …)` with `formatAssertion(\`bossSpawned (${bossType})\`, …)`.
- **`harness/validate/playthrough.mjs`**
  - In `writeFullArtifacts`, pass `bossType: preset.bossType` (and `preset` already passed) into `renderFindings`.
- **`harness/validate/verify-spire-ascent-artifacts.mjs`**
  - After reading `findings.md`, assert the boss-spawned line references `spire_warden` or `Summit Warden` and does not reference `annex_overseer`.
- **Writable output:** `game/validation/spire-ascent/**` regenerated by `pnpm validate:spire-ascent` — screenshots, `run-summary.json`, `probes.json`, `findings.md`, `console.log`.
- **Scope:** `harness/validate/**` and `game/validation/spire-ascent/**` only; no `game/server` gameplay changes.
