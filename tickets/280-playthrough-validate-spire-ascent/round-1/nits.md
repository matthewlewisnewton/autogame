## Spire Validation Findings Use Rooms Labels

`game/validation/spire-ascent/findings.md` was generated from a hard-coded Rooms/Training Caverns template: it titles the file "Rooms validation findings" and labels the boss assertion as `annex_overseer` even though the preset and run summary correctly validate `spire_warden`. This is non-blocking because the structured artifacts are correct, but it makes the human-facing validation report misleading.

### Acceptance Criteria
- Spire Ascent findings generated by `harness/validate/lib/findings.mjs` use a Spire-specific title and label the boss assertion with `spire_warden` or the active preset boss type.
