# Next Milestone Planning

Evaluate the current state of the project after the major refactoring and testing phase, and determine the next logical steps for development.

## Acceptance Criteria
- A markdown document is generated outlining the next 3 tickets to be added to the backlog.
- The proposed tickets address missing core features, polish, or expansions to more closely match the game design documents.
- The new tickets are added to `TASKS.md` and their respective folders/markdown files are created.

## Technical Specs
- **Files to modify**: A new planning artifact (e.g., `planning/next-milestone.md`) and updates to `TASKS.md`.
- **Considerations**: Review the original `CONTEXT.md` and `docs/design.md`. Potential areas to explore: advanced card combination mechanics (Z-Effect), multi-level dungeon progression, the trading/loot identification system (Tekker), or boss encounters.
- Always add ticket numbers that are consecutive (e.g. If the last ticket was #19, the next one should be #20).
- Each ticket should be fully independent and self-contained.
