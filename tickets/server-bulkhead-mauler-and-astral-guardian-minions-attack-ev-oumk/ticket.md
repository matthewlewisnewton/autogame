# Server: bulkhead_mauler and astral guardian minions attack every tick; mauler broadcasts CARD_USED 20x/sec

## Difficulty: easy

## Goal

The mauler branch in updateMinions (game/server/simulation.js:3083-3131) has no lastAttackAt/attackIntervalMs gating (unlike null_crawler 2000ms and storm_eagle 1500ms), so it swings its 9-damage cone every simulation tick (~180 DPS) and pushes a _pendingMinionBreaths entry per swing, which runGameLoopTick (game/server/index.js:1416-1421) broadcasts to the whole lobby as a CARD_USED event — 20 events/sec per mauler. Related: astral_guardian/aegis_sentinel default attackIntervalMs to one tick (game/server/cardEffects.js:208) because cardStats.json defines no interval, so they also hit per-tick (11 dmg x 20/s). Fix: add lastAttackAt/attackIntervalMs gating to the mauler branch and pick a sane default interval (1000-2000ms) for guardian melee minions. Found in code review 2026-06-09.

## Acceptance Criteria

- Mauler and guardian-type minions attack at a configured interval (>=1000ms) not per tick; CARD_USED minion-breath events are emitted at most once per attack; existing minion tests updated/passing
