**Root cause addressed:** `isPlayerOutOfCards()` only checks empty piles, so a hand with an uncastable MS spell (e.g. Signal Familiar at 25/99 MS) never triggers `checkRunTerminalState()`.

### Sub-tickets created

1. **01-detect-combat-exhaustion** — `canPlayerCastHandCard` + `isPlayerCombatExhausted` helpers and unit tests  
2. **02-auto-fail-on-combat-exhaustion** — grace timer, `tickCombatExhaustionGrace` in the game loop, wire into `checkRunTerminalState`  
3. **03-run-exhausted-debug-scenario** — update `run-exhausted` debug scenario to match the real MS stall + integration/regression tests
